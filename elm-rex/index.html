Currying

Currying is the process of translating the evaluation of a function that takes multiple arguments into evaluating a sequence of functions, each with a single argument. Source Wikipedia

Currying

          
            add : Int -> Int -> Int
            add number1 number2 = number1 + number2

            -- Let's curry it!

            add5 : Int -> Int
            add5 = add 5

            -- Let's use it now!

            marks : List Int
            bumpedMarks = List.map add5 marks

            add5 = (+) 5
            bumpedMarks = List.map ( (+) 5 ) marks

Currying - real example

          
            progression = List.map .score filteredMemberScores

              |> List.indexedMap (\index damage -> [index, damage] )

A sound type-system

  • Strong
  • Structural
  • No null
  • Implicit OR explicit

Strong: type always known

Structural: type defined by structure

No null: places where data can miss are known and handled with Maybe

Implicit or explicit: whatever fits the local needs

Records

          
            type alias Person = { name: String, dob: Date }

            -- Explicit typing
            getAge : Person -> Int

            -- Implicit typing
            getAge { dob } = ...

            -- Semi-implicit typing
            getAge : { whatever | dob: Date }
